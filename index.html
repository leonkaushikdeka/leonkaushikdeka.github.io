Make Each Repo Directly Usable

The showcase works best when every important repo has a visible next action. This is the publishing checklist to apply across the account.

Direct launch plan

  • Enable GitHub Pages for browser-only tools and static demos.
  • Deploy TypeScript apps to Vercel or Netlify with a demo account or fake data.
  • Publish Flutter APKs under GitHub Releases with screenshots and changelogs.
  • Publish Rust CLI binaries for Windows, Linux, and macOS releases.
  • Use Hugging Face Spaces or Streamlit for Python demos with sample data.

Trust and safety pass

  • Add privacy notes for tools that process documents, evidence, health, tax, or student data.
  • State prototype limits clearly for emergency, medical, security, and compliance projects.
  • Keep offensive security projects documentation-first and restricted to authorized lab use.
  • Do not publish secrets, tokens, production credentials, or real case data in demos.
  • Use stable README sections: Try, Install, Quick start, Status, Limitations, License.
