Publishing playbook.

The site is the front door. The repo-level work below is what makes the projects usable by someone who lands here for the first time.

DIRECT ACTION

Every repo needs one primary button

  • Try live demo
  • Download APK or binary
  • Deploy with sample data
  • Run locally in under five commands
README QUALITY

Make usage obvious above the fold

  • One-sentence value proposition
  • Screenshots or GIFs
  • Quick start and limitations
  • License and project status
TRUST AND SAFETY

Handle sensitive domains carefully

  • Privacy notes for document or case-data tools
  • Prototype disclaimers for emergency and compliance tools
  • Authorized-use framing for offensive security projects
  • No secrets, real case data, or unsafe public demos

Use it, fork it, or ask for a walkthrough.

Open an issue on GitHub for project-specific feedback, or send an email for collaboration, demos, responsible security discussion, and deployment review.
